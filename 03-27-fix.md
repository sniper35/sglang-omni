# Fix: Streaming voice cloning audio degradation

**Date:** 2025-03-27
**Issue:** sgl-project/sglang-omni#205 — voice cloning + streaming produces noise after first chunk

## Symptoms

1. After clicking Synthesize with streaming on, the user's reference audio appeared to play back with the target text content.
2. Streaming audio worked for ~2 chunks, then degraded to repeated "ssssss" segments.

## Root cause

### Bug 1 — reference audio confusion

`_dispatch` embedded the reference audio file as an inline `{"path": ref_audio, "mime_type": "audio/wav"}` dict in the chatbot user message. Gradio rendered this as a clickable audio player in the chat history. Because voice cloning produces output that sounds like the reference voice, users mistook the synthesized output for their own recording being played back.

### Bug 2 — streaming audio resets after prebuffer

The old `_streaming_path` was a single generator yielding `(chatbot_history, audio_chunk)` to **both** `gr.Chatbot` and `gr.Audio(streaming=True)` on every iteration:

```python
yield pending_history, (sr, audio_chunk)   # every chunk
```

Each yield triggered a chatbot component update. Gradio's re-render cycle reset the `gr.Audio(streaming=True)` component's internal accumulation buffer, so each chunk was played as an isolated fragment rather than appended to a continuous stream. The prebuffered first chunk (~1 s) sounded correct because it was self-contained; subsequent 0.5 s fragments started mid-waveform and produced audible noise.

An additional contributing factor: the initial `yield pending_history, None` sent `None` to the audio component before any data arrived, which could put it into a non-streaming state.

## Fix

Restructured `playground/tts/app.py` from a single dual-output generator into a three-step `.then()` event chain:

| Step | Function | Gradio outputs | Role |
|------|----------|---------------|------|
| 1 | `_prepare` | chatbot, gr.State | Validate input, build payload, show placeholder |
| 2 | `_synthesize` | **audio_output only** | Generator yielding `(sr, chunk)` tuples |
| 3 | `_finalize` | chatbot | Replace placeholder with WAV file + stats |

Key properties of the new design:

- The audio generator targets only `audio_output` — the chatbot is never updated in the same yield, so `gr.Audio(streaming=True)` accumulation is never disrupted.
- Reference audio is shown as text `"(voice clone reference provided)"` instead of an inline audio player.
- `None` is never sent to the audio component during streaming.
- `gr.State` passes the payload and accumulated samples between steps.

## Files changed

| File | Change |
|------|--------|
| `playground/tts/app.py` | Replaced dual-output generator with three-step `.then()` chain |
| `tests/test_playground_tts.py` | Updated lifecycle tests to cover `_prepare`, `_synthesize`, `_finalize` |
